# RaiseTimeLine — X風タイムラインSNSアプリ

X（旧Twitter）のタイムライン形式を参考にした、テキストベースのSNS風Webアプリです。
プログラミングスクール RaiseTech の課題として、講師の講義進行に合わせて学習目的で開発しています。

> アプリ名「RaiseTimeLine」は仮称です。
> 現在は**要件定義・設計フェーズ**です。実装開始後、起動手順・スクリーンショットを追記します。

---

## 主な機能

| 機能 | 内容 | Phase |
|------|------|-------|
| ユーザー登録・ログイン | メールアドレス + パスワード、JWT認証（アクセス + リフレッシュトークン） | 登録: 2 / ログイン: 1 |
| テキスト投稿 | 280文字制限。画像添付（1枚・JPEG/PNG・5MB以内、AWS S3 保存） | 1 |
| タイムライン表示 | 全ユーザーの投稿を新着順で表示。Phase 3 で「全体 / フォロー中」のタブ切替 | 1〜3 |
| いいね・コメント | 第三者の投稿にも可能。数の表示付き。二重いいねはDBのユニーク制約で防止 | 1 |
| 投稿の編集・削除 | 自分の投稿のみ（権限チェック） | 2 |
| ユーザープロフィール | アイコン（S3）・@ユーザー名・自己紹介・投稿一覧 | 2 |
| ユーザー検索 | ユーザー名・表示名で検索し、プロフィールへ遷移 | 2 |
| フォロー / フォロワー | プロフィールからフォロー。フォロー中タイムライン | 3 |

### X/Twitterとの差別化

- インプレッション数（表示回数）を**表示しない**
- リツイート（リポスト）機能を**用意しない**

## 開発ワークフロー

```
Issue作成 → ブランチ作成 → 実装 → ローカルで動作確認 → PR作成 → マージ → クリーンアップ
```

- ブランチ命名: `{type}/{説明}-#{Issue番号}`（例: `feature/timeline-api-#5`）
- コミットメッセージ: `{type}: {日本語説明}` + `Closes #番号`
- **main への直接プッシュ禁止**（ブランチ保護ルールで強制）・force push 禁止
- PRマージ後、リモートブランチは自動削除される設定
