# サーバー起動・ポート管理スキル

RaiseTimeLine プロジェクト（Spring Boot + Vite/React + PostgreSQL on Docker）のサーバー起動手順。

> **注意:** backend / frontend / docker-compose.yml は環境構築フェーズで作成される。
> 環境構築が終わるまで、このスキルの起動手順は実行できない（TaskManagement からの転用。構成は同一の前提）。

## 使用ポート（変更禁止）

| サーバー | ポート |
|---|---|
| バックエンド（Spring Boot） | **8080** |
| フロントエンド（Vite） | **5173** |
| データベース（PostgreSQL） | **5432** |

ポートを変えてはいけない理由：Vite のプロキシ設定と CORS 設定がこれらのポートを前提にしているため、別ポートで起動しても正常に連携できない。

## 起動前：ポート競合の確認と解消

起動前に必ず確認する。競合プロセスがあれば、停止してから起動する。別ポートへの切り替えや設定ファイルの書き換えは行わない。

```powershell
# 使用状況の確認
netstat -ano | findstr :8080
netstat -ano | findstr :5173
netstat -ano | findstr :5432

# 競合プロセスの停止（PIDを確認してから実行）
taskkill /PID <PID> /F
```

## 起動パターン

### 全サービス起動（通常の開発作業）

```powershell
# Step 1: データベース
docker-compose up -d

# Step 2: バックエンド（ポート8080）
cd backend
.\gradlew bootRun

# Step 3: フロントエンド（別ターミナル、ポート5173）
cd frontend
npm run dev
```

### フロントエンドのみ起動（UIのレイアウト・デザイン確認）

タイムライン取得など API に依存する機能はバックエンドなしでは動作しない。見た目の確認だけであればフロントのみで進められる。

```powershell
# ポート確認
netstat -ano | findstr :5173

# 起動（ポート5173固定）
cd frontend
npm run dev
```

## 禁止事項

- `8081`, `5174` など別ポートでの代替起動
- ポートを変えるための設定ファイル書き換え
- ポート競合を確認せずにそのまま起動を試みること
